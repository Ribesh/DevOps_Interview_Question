## Q. We want to launch centain pods only on specific nodes. These nodes should explicitly used only for this nodes. Is this possible to control deployments of such pods in the given scenario?

Answer:

-   Yes, this can be achieved by **Taints**

-   We can add a taint to a node
    ```bash
    kubeclt taint nodes <node_name> key=value:Effect
    ```

-   Taints are a property of nodes that push pods away if they don't tolerate the taint
    -   This means that the nodes must not accept any pods that doesn't tolerate all of the taints
    -   Like Labels, one or more Taints can be applied to a node


### Effects:

**NoSchedule** ‚Üí Pods that don‚Äôt tolerate the taint won‚Äôt be scheduled on the node.

**PreferNoSchedule** ‚Üí Kubernetes will try to avoid placing pods here, but may if necessary.

**NoExecute** ‚Üí New pods won‚Äôt schedule and existing non-tolerant pods will be evicted.


### Toleration (inside Pod spec)
```bash
tolerations:
- key: "key"
  operator: "Equal"
  value: "value"
  effect: "NoSchedule"
```
