# Pod is Stuck in CrashLoopBackOff



1. Check the Events of the Pods
```bash
kubectl describe pods <pod-name>
```

2. Check the logs
```bash
kubectl logs <pod-name> 
```

3. Check the previous terminated container logs
```bash
kubectl logs <pod-name> --previous
```

4. Check ConfigMaps and Secrets and configured correctly?



3. Misconfigured liveness probe

```bash
spec:
    containers:
        -   name: myapp
            image: myapp:latest
            livenessProve:
                httpGet:
                    path:   /healthz
                    port: 8080
                initialDelaySeconds: 5
                periodSeconds: 5
```
